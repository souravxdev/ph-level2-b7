# টাইপস্ক্রিপ্ট বেসিকস (TypeScript Basics) - কনসেপ্ট এবং সামারি

এই নোটে আমরা টাইপস্ক্রিপ্টের প্রাথমিক ধারণা, টাইপস, ফাংশন এবং বিভিন্ন গুরুত্বপূর্ণ ফিচার সম্পর্কে বিস্তারিত জানব। জাভাস্ক্রিপ্টের ডায়নামিক টাইপিংয়ের বদলে টাইপস্ক্রিপ্ট কীভাবে স্ট্যাটিক টাইপিং সুবিধা দেয়, সেটি একটি বাস্তবসম্মত "দোকান (Shop)"-এর উদাহরণ দিয়ে আলোচনা করা হলো।

## ১. টাইপস্ক্রিপ্ট পরিচিতি (Introduction to TypeScript)
জাভাস্ক্রিপ্ট একটি ডায়নামিক টাইপড ল্যাঙ্গুয়েজ, অর্থাৎ রানটাইমে ভেরিয়েবলের টাইপ পরিবর্তন করা সম্ভব। কিন্তু টাইপস্ক্রিপ্ট হলো স্ট্যাটিক টাইপড। এটি কম্পাইল করার আগেই টাইপ চেক করে এবং কোডে সম্ভাব্য ভুল (bugs) ধরতে সাহায্য করে।

- **এক্সটেনশন:** টাইপস্ক্রিপ্ট ফাইলের এক্সটেনশন হলো `.ts`।
- **কম্পাইলেশন:** মেশিন টাইপস্ক্রিপ্ট সরাসরি বুঝতে পারে মিঠা না, এটি প্রথমে জাভাস্ক্রিপ্টে (`.js`) রূপান্তরিত (compile) হয়। 

## ২. প্রিমিটিভ টাইপস (Primitive Types)
জাভাস্ক্রিপ্টের মতো টাইপস্ক্রিপ্টেও বেসিক কিছু ডেটা টাইপ রয়েছে। 

| টাইপ | ব্যবহার/উদাহরণ |
| --- | --- |
| `string` | টেক্সট বা স্ট্রিং ডেটার জন্য। যেমন: `let shopName: string = "রহিম স্টোর";` |
| `number` | যেকোনো ধরনের সংখ্যার জন্য। যেমন: `let totalProducts: number = 50;` |
| `boolean` | সত্য বা মিথ্যা নির্ধারণে। যেমন: `let isDiscountAvailable: boolean = false;` |

### **`null` বনাম `undefined`**
- **`null`:** এর মানে হলো ভ্যালুটি ইচ্ছাকৃতভাবে ফাঁকা রাখা হয়েছে (Empty value)। যেমন, কোনো ফিজিক্যাল দোকানের যদি অনলাইন রেটিং না থাকে, তবে তা `null` হতে পারে।
- **`undefined`:** এর মানে হলো ভ্যালুটির কোনো সংজ্ঞাই এখনো দেওয়া হয়নি (Not defined yet)।

## ৩. নন-প্রিমিটিভ টাইপস (Non-Primitive Types)
অ্যারে (Array) এবং অবজেক্ট (Object) হলো নন-প্রিমিটিভ টাইপস।

### অ্যারে টাইপিং (Array Typing)
অ্যারের মধ্যে কী ধরনের ডেটা থাকবে, তা টাইপস্ক্রিপ্টে নির্দিষ্ট করে দেওয়া যায়।
```typescript
let products: string[] = ["Rice", "Dal", "Oil", "Salt"];
let prices: number[] = [60, 120, 200, 30];
```

### অবজেক্ট টাইপিং এবং অপশনাল প্রপার্টি (Object Typing & Optional Properties)
অবজেক্টের প্রতিটি প্রপার্টির টাইপ আগে থেকে নির্ধারণ করা যায়। কোনো প্রপার্টি যদি বাধ্যতামূলক না হয়, তবে `?` (কোয়েশ্চন মার্ক) ব্যবহার করে সেটিকে **অপশনাল** করা যায়।
```typescript
let product: {
  name: string;
  price: number;
  category: "Grocery" | "Electronics"; // String Literal Type
  discount?: number; // Optional Property
} = {
  name: "Rice",
  price: 60,
  category: "Grocery"
};
```

## ৪. ফাংশন এবং ভয়েড টাইপ (Functions & Void Type)
টাইপস্ক্রিপ্টে ফাংশনের প্যারামিটার এবং রিটার্ন টাইপ স্পষ্টভাবে বলে দেওয়া যায়। 

- **প্যারামিটার ও রিটার্ন টাইপ:**
```typescript
function calculateTotal(price: number, quantity: number): number {
  return price * quantity;
}
```
- **ভয়েড (Void) টাইপ:** যদি কোনো ফাংশন থেকে কোনো কিছু রিটার্ন করা না হয় (যেমন শুধু `console.log` করা হয়), তখন তার রিটার্ন টাইপ হয় `void`।
```typescript
function showMessage(): void {
  console.log("This is a void function. It returns nothing.");
}
```

## ৫. রেস্ট, স্প্রেড অপারেটর এবং ডিস্ট্রাকচারিং (Rest, Spread & Destructuring)
- **রেস্ট অপারেটর (`...`):** একাধিক আর্গুমেন্টকে একটি অ্যারেতে রূপান্তর করতে ফাংশন প্যারামিটারে ব্যবহৃত হয়।
```typescript
function calculateCartTotal(...prices: number[]): number {
  return prices.reduce((sum, currentPrice) => sum + currentPrice, 0);
}
```
- **স্প্রেড অপারেটর (`...`):** অ্যারে বা অবজেক্টকে ভেঙে অন্য অ্যারে বা অবজেক্টের সাথে যুক্ত করতে ব্যবহৃত হয়।
```typescript
let baseProducts: string[] = ["Rice", "Dal"];
let newProducts: string[] = [...baseProducts, "Oil", "Salt"];
```
- **ডিস্ট্রাকচারিং (Destructuring):** অবজেক্ট থেকে নির্দিষ্ট প্রপার্টি বের করে আনার পদ্ধতি।
```typescript
const buyer = { customerName: "Karim", contactNumber: 1700000000, address: "Dhaka" };
const { customerName, contactNumber } = buyer;
```

## ৬. টাইপ এলিয়াসিং (Type Aliasing)
বারবার বড় অবজেক্ট টাইপ না লিখে, `type` কি-ওয়ার্ড ব্যবহার করে একটি কাস্টম টাইপ বা "ডাকনাম" তৈরি করা যায়। টাইপের নামকরণের ক্ষেত্রে শুরুতে বড় হাতের `T` (যেমন: `TProduct`) ব্যবহার করা একটি ভালো কনভেনশন।
```typescript
type TProduct = {
  productName: string;
  price: number;
};

const item: TProduct = {
  productName: "Water Bottle",
  price: 30
};
```

## ৭. ইউনিয়ন এবং ইন্টারসেকশন (Union & Intersection)
- **ইউনিয়ন (`|`):** এর মানে হলো "অথবা (OR)"। একাধিক টাইপের মধ্যে যেকোনো একটি হতে পারে।
```typescript
type TPaymentMethod = "Bikash" | "Nagad" | "Rocket";
let payment: TPaymentMethod = "Bikash"; // শুধু এই তিনটির একটি হবে
```
- **ইন্টারসেকশন (`&`):** এর মানে হলো "এবং (AND)"। একাধিক টাইপকে যুক্ত করে একটি নতুন টাইপ তৈরি করে।
```typescript
type TUser = { name: string };
type TSeller = TUser & { shopName: string };

const seller: TSeller = {
  name: "Rahim",
  shopName: "Rahim Store"
};
```

### 💡 সার্বিক উন্নতি ও পরামর্শ (Overall Enhancement Suggestion)
আপনার শেখার ধরন এবং নোট অনুযায়ী আপনার বেসিক ফাউন্ডেশন বেশ শক্তিশালী হচ্ছে। তবে এই বিষয়গুলোকে পুরোপুরি আয়ত্ত করতে আপনাকে নিচের বিষয়গুলো নিয়ে একটু পড়াশোনা ও প্র্যাকটিস করতে হবে:
1. **Interfaces বনাম Type Aliases:** টাইপ এলিয়াস শিখেছেন, কিন্তু অবজেক্ট ও ক্লাসের ক্ষেত্রে `interface` কখন এবং কেন ব্যবহার করা উচিত, তা জানা জরুরি।
2. **Never এবং Unknown টাইপ:** ভয়েড (Void) টাইপ দেখেছেন, এর পাশাপাশি `never` এবং `unknown` টাইপের ব্যবহারিক পার্থক্যগুলো জেনে নেওয়া উচিত (যা হয়তো ক্লাসের পরবর্তী অংশে আলোচনা করা হয়েছে)।
3. **Generics (জেনেরিকস):** রিউজেবল এবং স্কেলেবল কোড লেখার জন্য জেনেরিকস হলো টাইপস্ক্রিপ্টের অন্যতম শক্তিশালী ফিচার। সামনে এটি নিয়ে বিস্তারিত জানলে টাইপস্ক্রিপ্টের আসল ক্ষমতা বুঝতে পারবেন।

### 📝 সারাংশ (Summary)
- টাইপস্ক্রিপ্ট জাভাস্ক্রিপ্টের উপর একটি স্ট্যাটিক টাইপিং লেয়ার, যা রানটাইমের আগেই এরর ধরতে সাহায্য করে।
- `string`, `number`, `boolean` ইত্যাদি হলো প্রিমিটিভ টাইপস। `null` এবং `undefined`-এর মধ্যে মৌলিক পার্থক্য রয়েছে।
- অ্যারে এবং অবজেক্টের টাইপ নির্দিষ্ট করা যায়, এবং `?` ব্যবহার করে প্রপার্টি অপশনাল করা যায়।
- ফাংশনের প্যারামিটার ও রিটার্ন টাইপ ডিফাইন করা যায়। কিছু রিটার্ন না করলে তা `void` হয়।
- `type` কি-ওয়ার্ড দিয়ে কাস্টম টাইপ বানানো যায় (Type Aliasing)।
- `|` (Union) দিয়ে "যেকোনো একটি" এবং `&` (Intersection) দিয়ে "সবগুলোর সমন্বয়" বোঝানো হয়।

### 🎯 ইন্টারভিউ প্রস্তুতি (Interview Preparation)
এই মডিউলের কনসেপ্টগুলোর ওপর ভিত্তি করে ৫-৬টি গুরুত্বপূর্ণ ইন্টারভিউ প্রশ্ন ও তার উত্তর দেওয়া হলো:

**১. প্রশ্ন: টাইপস্ক্রিপ্টে `null` এবং `undefined` এর মধ্যে পার্থক্য কী?**
**উত্তর:** `undefined` মানে হলো একটি ভেরিয়েবল ডিক্লেয়ার করা হয়েছে কিন্তু এখনো কোনো ভ্যালু অ্যাসাইন করা হয়নি (Not initialized)। অন্যদিকে, `null` হলো একটি অ্যাসাইনমেন্ট ভ্যালু, যা নির্দেশ করে যে ভেরিয়েবলটি ইচ্ছাকৃতভাবে ফাঁকা বা এর কোনো মান নেই (Intentional empty value)। 

**২. প্রশ্ন: টাইপস্ক্রিপ্টে `void` টাইপ বলতে কী বোঝায়? এটি কখন ব্যবহার করা হয়?**
**উত্তর:** `void` টাইপ নির্দেশ করে যে একটি ফাংশন থেকে কোনো ভ্যালু রিটার্ন হবে না। সাধারণত যেসব ফাংশন শুধু কনসোলে কিছু প্রিন্ট করে বা বাইরের কোনো স্টেট পরিবর্তন করে (কিন্তু কিছু রিটার্ন করে না), তাদের রিটার্ন টাইপ `void` দেওয়া হয়।

**৩. প্রশ্ন: Type Alias কী এবং কেন ব্যবহার করা হয়?**
**উত্তর:** Type Alias (`type` কি-ওয়ার্ড) ব্যবহার করে বিদ্যমান কোনো টাইপ বা বড় অবজেক্ট টাইপের জন্য একটি নতুন নাম (কাস্টম টাইপ) তৈরি করা যায়। এটি কোডকে আরও রিডেবল (readability) করে এবং একই বড় টাইপ বারবার লেখার ঝামেলা দূর করে (reusability)।

**৪. প্রশ্ন: Union Type (`|`) এবং Intersection Type (`&`) এর মধ্যে পার্থক্য কী?**
**উত্তর:** Union Type (`|`) নির্দেশ করে যে একটি ভেরিয়েবল একাধিক টাইপের যেকোনো একটি হতে পারে (OR logic)। আর Intersection Type (`&`) একাধিক টাইপকে একত্রিত করে একটি নতুন টাইপ তৈরি করে, যেখানে পূর্বের সব টাইপের প্রপার্টি থাকা বাধ্যতামূলক (AND logic)।

**৫. প্রশ্ন: অপশনাল প্রপার্টি (Optional Property) কীভাবে ডিক্লেয়ার করতে হয় এবং এর সুবিধা কী?**
**উত্তর:** অবজেক্ট টাইপ বা ইন্টারফেসে প্রপার্টির নামের শেষে একটি কোয়েশ্চন মার্ক (`?`) ব্যবহার করে অপশনাল প্রপার্টি ডিক্লেয়ার করা হয় (যেমন: `discount?: number`)। এর সুবিধা হলো, অবজেক্ট তৈরি করার সময় এই প্রপার্টিটি প্রদান করা বাধ্যতামূলক থাকে মহাশয় না, ফলে ফ্লেক্সিবিলিটি বাড়ে।

**৬. প্রশ্ন: String Literal Type বলতে কী বোঝায়?**
**উত্তর:** String Literal Type হলো এমন একটি টাইপ, যেখানে কোনো ভেরিয়েবলের মান হিসেবে শুধুমাত্র নির্দিষ্ট কিছু স্ট্রিং ভ্যালুকেই অ্যালাউ করা হয়। যেমন `type TDirection = "Left" | "Right";`। এখানে "Left" বা "Right" ছাড়া অন্য কোনো স্ট্রিং অ্যাসাইন করতে গেলে টাইপস্ক্রিপ্ট এরর দিবে।
